### Main Entity Tables

These tables represent the core, independent objects of the application.

| Table Name | One-Sentence Explanation |
| :--- | :--- |
| **user** | Stores all user account information, including credentials, roles, and preferences. |
| **chat_session** | Represents a single, continuous conversation thread between a user and an AI persona. |
| **chat_message** | Contains a single message within a chat, including its content, sender, and associated metadata. |
| **persona** | Defines a configurable AI assistant with a specific personality, knowledge, and set of tools. |
| **prompt** | Holds a specific instruction template used to guide the LLM in generating a response. |
| **tool** | Defines a custom function or capability that an AI persona can execute to perform actions. |
| **document** | Stores metadata for a single piece of content (like a file or webpage) that has been ingested into the system. |
| **connector** | Defines a configuration for a data source, such as a website or a document repository, to be indexed. |
| **credential** | Securely stores the authentication credentials required to access data sources defined by connectors. |
| **document_set** | Represents a logical grouping of documents that can be used as a bounded knowledge base for a persona. |
| **tag** | Stores a single key-value metadata label that can be applied to documents for filtering and organization. |
| **api_key** | Contains API keys that grant programmatic access to the Onyx system on behalf of a user. |
| **search_settings** | Holds the configuration for the embedding and reranking models that power the search functionality. |
| **notification** | Stores system-generated messages and alerts intended to be displayed to users in the UI. |
| **kg_entity** | Represents a single, distinct real-world entity (like a person or company) extracted from documents. |
| **kg_entity_type** | Defines a class or category for knowledge graph entities (e.g., "Person", "Location"). |
| **kg_relationship_type** | Defines a type of connection or relationship that can exist between two knowledge graph entities. |
| **index_attempt** | Logs a single execution of an indexing job, tracking its status, progress, and any errors. |
| **tool_call** | Records a specific instance of an AI agent executing a tool during a chat conversation. |
| **search_doc** | A snapshot of a document as it was retrieved in a search, used to preserve the context of past chat messages. |
| **access_token** | Stores user session tokens for database-backed authentication. |
| **oauth_account** | Links a user's main account to an external identity from an OAuth provider like Google. |
| **standard_answer** | A pre-defined, canonical answer to a common question. |
| **standard_answer_category** | A category for grouping related standard answers. |
| **slack_bot_config** | Configuration for the main Slack bot integration. |
| **slack_channel_config** | Custom configurations for the Slack bot on a per-channel basis. |
| **user_file** | Represents a file uploaded directly by a user. |
| **user_folder** | A folder created by a user to organize their files. |
| **chat_folder** | A folder created by a user to organize their chat sessions. |
| **chunk_stats** | Stores statistics about individual document chunks, such as information content boosts. |
| **sync_record** | Tracks the status of a synchronization operation, such as updating a document set in the search index. |
| **background_error** | Logs errors that occur during background jobs, linked to the relevant connector pair. |
| **index_attempt_error** | Logs specific errors that occurred during a particular index attempt. |
| **kg_term** | Stores terms and their associated entity types for knowledge graph searching. |
| **kg_relationship** | Represents a specific, discovered relationship instance between two `kg_entity` records. |
| **kg_relationship_extraction_staging** | A temporary staging area for new knowledge graph relationships before they are finalized. |
| **kg_entity_extraction_staging** | A temporary staging area for new knowledge graph entities before they are finalized. |
| **kg_relationship_type_extraction_staging** | A temporary staging area for new knowledge graph relationship types. |
| **embedding_provider** | Stores details about different embedding model providers. |
| **cloud_embedding_provider** | Specific configuration for cloud-based embedding providers. |
| **llm_provider** | Stores details about different Large Language Model providers. |
| **custom_llm_provider** | Configuration for custom or self-hosted LLM providers. |
| **reranker_provider** | Stores details about different reranker model providers. |
| **input_prompt** | A prompt provided by a user that can be saved and reused. |
| **agent** | Defines a configurable agent with specific skills and behaviors. |
| **agent_sub_query** | A sub-query generated by an agent as part of a larger task. |
| **agent_search_metrics** | Metrics related to the performance of agent searches. |
| **agent_tool** | A tool specifically assigned to an agent. |
| **agent_persona** | A persona specifically designed for an agent. |
| **agent_llm** | An LLM configuration specifically for an agent. |
| **agent_prompt** | A prompt template specifically for an agent. |
| **agent_chat_session** | A chat session that is handled by an agent. |
| **agent_chat_message** | A message within an agent-handled chat session. |
| **agent_tool_call** | A record of a tool call made by an agent. |
| **agent_feedback** | Feedback provided by a user on an agent's performance. |
| **agent_user** | A user that is specifically designated as an agent. |
| **agent_group** | A group of agents. |
| **agent_group_user** | A user that is a member of an agent group. |
| **agent_group_agent** | An agent that is a member of an agent group. |
| **agent_group_permission** | Permissions associated with an agent group. |
| **agent_permission** | Permissions associated with a specific agent. |
| **user_group** | A group of users. |
| **user_group_user** | A user that is a member of a user group. |
| **user_group_permission** | Permissions associated with a user group. |
| **permission** | A specific permission that can be granted to users or groups. |
| **role** | A role that can be assigned to a user, with a set of associated permissions. |
| **role_permission** | The link between a role and a specific permission. |
| **user_role** | The link between a user and a specific role. |
| **external_user** | A user that is managed by an external identity provider. |
| **external_user_group** | A group of external users. |
| **external_user_group_user** | An external user that is a member of an external user group. |
| **external_user_group_permission** | Permissions associated with an external user group. |
| **external_user_permission** | Permissions associated with a specific external user. |
| **external_user_role** | The link between an external user and a specific role. |
| **external_user_group_role** | The link between an external user group and a specific role. |

### Association (Many-to-Many) & Index Tables

These tables exist to link main entities together or to provide indexes for faster lookups.

| Table Name | One-Sentence Explanation |
| :--- | :--- |
| **alembic_version** | Tracks the current database schema version for Alembic migrations. |
| **alembic_version_tenant** | Tracks schema versions for individual tenants in a multi-tenant setup. |
| **connector_credential_pair** | Links a `connector` to a `credential`, forming a runnable indexing job. |
| **document_by_connector_credential_pair** | Links a document to the specific job that indexed it, for tracking purposes. |
| **document__tag** | Links documents to their assigned metadata tags. |
| **persona__document_set** | Links personas to the specific document sets they are allowed to access. |
| **persona__prompt** | Links personas to the specific prompt templates they use. |
| **persona__tool** | Links personas to the specific tools they are allowed to use. |
| **persona__user** | Links personas to the users who have access to them. |
| **document_set__user** | Manages which users have access to which document sets. |
| **document_set__connector_credential_pair** | Links document sets to the specific indexing jobs that populate them. |
| **chat_message__search_doc** | Links a specific chat message to the search documents that were used as citations. |
| **chat_message__standard_answer** | Links a chat message to a pre-defined standard answer. |
| **agent_sub_query__search_doc** | Links an agent's sub-query to the search documents it retrieved. |
| **standard_answer__standard_answer_category** | Links standard answers to their respective categories. |
| **slack_channel_config__standard_answer_category** | Links a Slack channel's configuration to the standard answer categories it can use. |
| **chat_feedback** | Stores user feedback (e.g., upvotes/downvotes) on a specific chat message. |
| **document_retrieval_feedback** | Stores user feedback on the relevance of a specific document retrieved in a search. |
